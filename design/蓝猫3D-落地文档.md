## 1. 产品定位

**蓝猫3D = 工具榜单（Leaderboard）+ 九步产线实战课（The Course）。**

- **价值主张**：把主流 AI 3D 工具按角色产线的九道工序逐个实测、排名——告诉你每一步交给谁最快、哪一步得自己练。看懂榜单，再跟蓝猫的课走一遍，做出一个能进 Unity / UE5 的成品角色。
- **核心洞察（贯穿全站的叙事）**：难度全压在中段——**两头（生成、导出）AI 最省力，中段（雕刻 03 / 重拓扑 04 / 绑定 08）AI 收不了的尾，正是蓝猫带你练的地方。**
- **人设 / Mentor**：蓝猫 · BlueCat —— 数字人产品经理 / `bluecatbot.com` 主理人 / B 站创作者。「教程到处是，但从概念到能进引擎的角色，没有一条能一口气走通的路。这门课，就是我希望当年有人直接塞给我的那条完整路径。」
- **商业模式**：内容（榜单 + 九步产线）免费引流 → 实战课付费转化（散养 ¥999 / 亲传 ¥3999）。

### 5.2 工具榜单矩阵

> 评测方法：**同一提示图 · 盲测 · 蓝猫复核**。「告诉你每一步交给谁最快、哪一步得自己练」。

| 工序 | AI/手艺 | 候选工具与一句话评语 |
|---|---|---|
| 概念设定 | AI | **Nano Banana Pro**（角色身份锁定、跨图一致最强）· ChatGPT（设定 + prompt） |
| 模型生成（图生模型） | AI | **混元 3D v3.1**（图生一流 · 拓扑干净 · 中文友好）· Meshy 6（通用 · 多视图稳）· Tripo（细节强 · 偏慢） |
| 雕刻 | 手艺 | Blender 4.5 LTS（清理瑕疵 · 修解剖 · 定剪影——靠手感，蓝猫带练） |
| 重拓扑 / 优化 | 手艺 | RetopoFlow / 手动（手动可控 · 上限高）· QuadRemesher（自动快 · 需手修）· QuadriFlow（够用 · 偏保守） |
| UV 展开 | AI+手修 | Blender（自动展开能用，缝线/island/密度要手修）· UVPackmaster / Zen UV（打包榨空间） |
| 纹理烘焙 | 手艺 | Marmoset Toolbag · Blender · Substance Painter（cage/射线/切线空间踩对参数才出干净结果） |
| 上色 + 材质（纹理 PBR） | AI+手修 | **Substance Painter**（手动上限高）· Meshy（一键 PBR · 素材全，做底色）· 重绘强可补绘 |
| 绑定 / 动画 | 手艺 | Blender Weight Paint（权重可控 · 经典）· AccuRIG 2 / Mixamo（自动绑定 · 快 · 变形一般）· Meshy/Tripo 自动绑（直接能用 · 偏通用） |
| 导出到引擎 | AI/标准 | Unity 6 / UE5（规范导出 · 贴图对齐 · 预览测试）· Blender（导出 GLB/FBX） |

> 榜单 UI 要点：每工序一行，工具按推荐度排序，每个工具配「优势 · 适用」标签（如 `图生一流 · 拓扑干净`），并标注该工序属于 **AI 自动** 还是 **手艺活**（呼应「中段最难」叙事）。

### 5.4 FAQ（`faq.json`）

1. **需要 3D 或编程基础吗？** 不需要。课程为零基础设计，每一步从头讲起；有基础的会走得更快、挖得更深。
2. **电脑要什么配置？** 能跑 Blender 和基础 AI 工具的近几年电脑就够。独立显卡推荐但非必需。
3. **工具订阅要花多少钱？** 课里工具大多免费或有充足免费额度；可选付费档通常每月 10–30 美元上下。
4. **学完要多久？** 内容相当于 5 天体系化课程，多数人约一周修完，之后边做项目边回看。
5. **学完能接活吗？** 能。课程奔着「从零做出一个能吸引真实客户的作品集」设计。

```css

  --abyss:#090D18; --night:#0F1626; --surface:#161F35; --surface-2:#1C2742;
  /* 线 */
  --line:rgba(110,150,220,.16); --line-2:rgba(110,150,220,.32);
  /* 文字 */
  --text:#E9EEFA; --muted:#90A0C0; --faint:#5E6E92;
  /* 强调 */
  --cobalt:#5B7CFA; --glow:#37E1FF; --gold:#F5C24E;
  --red:#F0685B; --green:#54C99A;
}
```

- **字体**：标题 `Sora` / `Space Grotesk` + `Noto Sans SC`（700/800）；技术标签 `JetBrains Mono`；正文 `Inter` / `Noto Sans SC`。

## 7. 技术落地方案

### 7.1 技术选型

| 维度 | 选型 | 理由 |
|---|---|---|
| 站点类型 | **数据驱动静态站**（Python 构建脚本生成静态 HTML） | 内容无动态需求；对齐工作区 vlog-blog / svgsafe / learn 范式；SEO 友好；部署最省 |
| 构建 | `build_site.py`（Jinja-less 字符串模板或轻量模板） | 同 vlog-blog；`*.json` + 模板 + 固定 `style.css`/`i18n.js` 保证三语 × 9 页一致 |
| 前端 | 原生 HTML/CSS/JS（无框架） | 内容站够用；设计稿已是原生，零迁移成本 |
| 三语 | `i18n.js` + URL 路由 `/zh//en//ja/`，一语一文件 | 同 learn / svgsafe 已验证范式 |
| 课程支付（Phase 3） | `stripe-payments` skill（一次性付费 + webhook 履约） | 红线：密钥只进服务器 `.env`，绝不进 git/聊天 |
| 部署 | `bluecat-deploy` skill（静态站 → nginx root + certbot） | `3d.bluecatbot.com`，chi 服务器 |

### 7.3 数据模型（`modules.json` 单条 schema）

每个模块对象（落地稿字段已齐全，直接灌）：

```json

  "tools": ["ChatGPT", "Nano Banana Pro"],
  "why": "下游图生 3D 是垃圾进垃圾出……",   // 「为什么这步定生死」段落
  "hardRules": ["A-pose 四肢分离……", "中性平光……", "..."],
  "substeps": [
    { "tag": "1.1", "title": "立意·先用文字把角色钉死", "body": "...", "prompt": "..." }
  ],
  "styleBranches": [ {"name":"写实","tip":"...","code":"..."} ],   // 模块01有
  "pitfalls": ["用了戏剧光 → 阴影被烤死……", "..."],
  "deliverables": ["1 张定稿正视图……", "..."],
```

`leaderboard.json` / `course.json` / `faq.json` 直接对应 §5.2 / §5.3 / §5.4 的表结构。

### 7.4 构建管线

```
build_site.py:
  for lang in [zh, en, ja]:
    render home.html  ← leaderboard.json + modules(总览) + course.json + faq.json + i18n[lang]
    for m in modules.json:
      render module.html ← m + i18n[lang]   → out/<lang>/pipeline/<num>.html
  copy static/ → out/
  apex(/) → 302 redirect → /zh/   （由 nginx/Caddy 配）
```

- 每语 × (1 首页 + 9 模块 + 1 榜单 + 1 课程) ≈ 12 页 → 三语共 ~36 页静态产物。
- 固定模板 + 固定 `style.css` 保证「9 页 × 3 语」风格、版式、字体、颜色完全一致。

### 7.5 三语化

## 9. 部署方案（`3d.bluecatbot.com`）

走 `bluecat-deploy` skill（静态站路径）：

1. **子域名**：`3d.bluecatbot.com`（端口仅本地构建无需，静态站走 nginx root）。

3. **上线**：`out/` → 服务器 nginx root；nginx 新增 `3d` 子域名 server block + certbot SSL。
   - ⚠️ 复用记忆教训：nginx 静态站 404 常因目录权限——上线后 `chmod -R o+rX` 站点目录。
   - 确认部署目标服务器（chi `57.181.215.147` 还是游戏服 `54.248.150.247`/Caddy）——**以 `servers.json` 为准**，建站时落 `servers.json`（gitignore）。

## 12. 风险与开放问题

1. **落地页是 6.7MB 打包态**——必须先去打包还原可维护源码，否则无法迭代。这是 P0 的隐藏工作量。
2. **两套设计令牌**（模块页 vs 落地页）——P0 统一为一套（§6），避免上线后视觉割裂。
3. **部署目标服务器待确认**——bluecatbot 子域名既有走 chi（nginx）也有走游戏服（Caddy）的先例；建站时定，落 `servers.json`。
4. **榜单数据时效**——AI 3D 工具（混元/Meshy/Tripo 版本）迭代极快，榜单需标注「实测日期」并定期复测，否则很快过时。建议榜单条目带 `testedAt` 字段。
5. **课程交付与履约**——视频托管在哪（B 站不付费墙友好？自托管？）、社群承接、亲传档席位计数，都是 P4 前要定的运营问题，非纯工程。
6. **「亲传 ¥3999 / 每月 10 席」库存**——需服务端状态（最简：当月计数器 + 售罄置灰 CTA）。
